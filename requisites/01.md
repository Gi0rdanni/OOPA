Prezados alunos,
Após a nossa aula sobre arquitetura de software em camadas, chegou o momento de colocar o conhecimento em prática! A tarefa a seguir consiste na implementação de uma pequena aplicação de Cadastro e Listagem de Itens. O objetivo principal é que vocês apliquem os conceitos da arquitetura em camadas, organizando o código de forma clara, coesa e com baixo acoplamento entre os componentes.


1) Arquitetura em Camadas
A estrutura do projeto de vocês deve obrigatoriamente seguir a seguinte separação de responsabilidades:
Model: Será a camada responsável por representar a entidade do nosso domínio. Nesta camada, teremos a classe Item com seus atributos.
DAO (Data Access Object): Esta camada será a única responsável pelo acesso e manipulação dos dados. Ela fará a ponte entre a nossa aplicação e o "banco de dados" (que para este exercício pode ser simulado com uma estrutura de dados em memória, como uma lista ou um dicionário).
Controller: A camada de controle é o cérebro da nossa aplicação. Ela será responsável por receber as requisições da View, processar a lógica de negócio e coordenar as ações entre a View e o DAO.
View: Esta é a camada de apresentação, responsável pela interação com o usuário. Ela exibirá os dados e capturará as ações do usuário, repassando-as para a Controller.


2) Requisitos da Aplicação
21) Camada Model
Criar uma classe Item com os seguintes atributos:
id (inteiro, identificador único)
descricao (string)
quantidade (inteiro)

22) Camada DAO
Criar uma classe ItemDAO com os seguintes métodos:
adicionar(item: Item): Adiciona um novo item à estrutura de dados.
listarTodos() -> list[Item]: Retorna a lista de todos os itens cadastrados.

23) Camada Controller
Criar uma classe ItemController com os seguintes métodos, que farão a interface com a View:
criarItem(nome: string, descricao: string, quantidade: int)
obterTodosOsItens() -> list[Item]

24) Camada View
Aplicação com StreamLit.

25) Banco de dados
Usar o banco de dados SQLIte para armazenamento de dados.


3) O que será avaliado?
Organização do código: A correta separação das classes em suas respectivas camadas.
Coesão e Acoplamento: A View não deve conhecer o DAO, e o Model não deve ter lógica de negócio ou acesso a dados. O Controller deve orquestrar as interações.
Funcionalidade: A aplicação deve realizar as suas operações: Criar e listar Itens.
Boas práticas de programação: Clareza no código, nomes de variáveis e métodos significativos.
